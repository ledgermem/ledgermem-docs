`llama-index-memory-ledgermem` exposes LedgerMem as both a vector store and
a chat memory for LlamaIndex.

## Install

```bash
pip install llama-index-memory-ledgermem
```

## Vector store

```python
from llama_index.core import VectorStoreIndex, StorageContext
from llama_index_memory_ledgermem import LedgerMemVectorStore

vector_store = LedgerMemVectorStore.from_env(namespace="docs")
storage_context = StorageContext.from_defaults(vector_store=vector_store)

index = VectorStoreIndex.from_documents(
    documents,
    storage_context=storage_context,
)

query_engine = index.as_query_engine(similarity_top_k=5)
print(query_engine.query("What is the deployment process?"))
```

The vector store implements the LlamaIndex `BasePydanticVectorStore`
interface, so all the usual goodies — metadata filters, hybrid search,
streaming responses — work without modification. Embeddings are still
managed by LedgerMem; LlamaIndex's own embedding model is ignored.

## Chat memory

```python
from llama_index.core.chat_engine import SimpleChatEngine
from llama_index.llms.openai import OpenAI
from llama_index_memory_ledgermem import LedgerMemChatMemory

chat_memory = LedgerMemChatMemory(user_id="u_123")

engine = SimpleChatEngine.from_defaults(
    llm=OpenAI(model="gpt-4o-mini"),
    memory=chat_memory,
)

print(engine.chat("What did I say I was working on?"))
```

`LedgerMemChatMemory` implements the LlamaIndex `BaseMemory` interface and
uses LedgerMem as the durable backing store. Each user message is checked
against existing memories; novel facts are stored automatically. Retrieval
runs on every turn so the LLM sees relevant prior memories as system
context.

## Filters

```python
from llama_index.core.vector_stores import MetadataFilter, MetadataFilters

filters = MetadataFilters(filters=[
    MetadataFilter(key="user_id", value="u_123"),
    MetadataFilter(key="source", value="onboarding"),
])

retriever = index.as_retriever(filters=filters, similarity_top_k=5)
nodes = retriever.retrieve("when was the user onboarded?")
```

## Why use LedgerMem with LlamaIndex?

- You get a managed vector store with no infra to run.
- You get hybrid retrieval (vector + BM25 + rerank) for free.
- You can swap the embedding model later without re-ingesting your corpus.
- Multi-tenant isolation is enforced by workspace, not by a metadata
  filter you have to remember to add.
