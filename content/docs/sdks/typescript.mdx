The TypeScript SDK is the most commonly used client for LedgerMem and the
reference implementation for every other SDK.

- npm: [`ledgermem`](https://www.npmjs.com/package/ledgermem)
- GitHub: [`ledgermem/ledgermem-js`](https://github.com/ledgermem/ledgermem-js)
- Runtimes: Node 18+, Bun 1+, Deno 1.40+, modern browsers (with caveats — see below).

## Install

```bash
npm install ledgermem
# or
pnpm add ledgermem
# or
bun add ledgermem
```

## Configure

```ts
import { LedgerMem } from "ledgermem";

const lm = new LedgerMem({
  apiKey: process.env.LEDGERMEM_API_KEY!,
  workspaceId: process.env.LEDGERMEM_WORKSPACE_ID!,
  // Optional:
  baseUrl: "https://api.proofly.dev",
  timeoutMs: 30_000,
  retries: { max: 2, backoff: "exponential" },
  fetch: globalThis.fetch,
});
```

The constructor accepts a custom `fetch` so you can run inside Cloudflare
Workers or Vercel Edge runtimes, where the global `fetch` shape differs
slightly.

## Memories

```ts
// Create
const memory = await lm.memories.create({
  content: "User prefers dark mode.",
  metadata: { user_id: "u_123" },
  kind: "semantic",
});

// Read
const fetched = await lm.memories.get(memory.id);

// Update
await lm.memories.update(memory.id, {
  metadata: { user_id: "u_123", updated_by: "agent" },
});

// Delete (soft by default)
await lm.memories.delete(memory.id);

// Hard delete
await lm.memories.delete(memory.id, { hard: true });

// List with cursor pagination
for await (const page of lm.memories.list({ filter: { user_id: "u_123" } })) {
  for (const m of page.data) {
    console.log(m.id, m.content);
  }
}
```

## Search

```ts
const results = await lm.search({
  query: "what does the user like in their editor?",
  limit: 5,
  strategy: "rerank",
  filter: { user_id: "u_123" },
});

for (const hit of results.results) {
  console.log(hit.score, hit.content);
}
```

## Streaming

For very large result sets, use the streaming search endpoint:

```ts
for await (const hit of lm.search.stream({ query: "...", limit: 200 })) {
  console.log(hit.id, hit.score);
}
```

## Error handling

All API errors throw `LedgerMemError`, which carries the HTTP status, the
server-side request ID, and any structured error payload:

```ts
import { LedgerMemError } from "ledgermem";

try {
  await lm.memories.get("mem_does_not_exist");
} catch (err) {
  if (err instanceof LedgerMemError && err.status === 404) {
    console.log("not found:", err.requestId);
  } else {
    throw err;
  }
}
```

## Browser usage

You can use the SDK in a browser, but never with a long-lived API key
embedded in the bundle. Instead, mint a temporary, scoped key from your
backend and pass it to `new LedgerMem({ apiKey })`. The
[playground](https://play.proofly.dev) does exactly this — see its
`/api/key` route for a working example.

## TypeScript types

Every API shape is exported. The most useful:

```ts
import type {
  Memory,
  SearchHit,
  SearchRequest,
  SearchStrategy,
  ListResponse,
} from "ledgermem";
```
